# Deep Insights Copilot: CTO Architecture Overview

## Executive Summary
The Deep Insights Copilot is a highly secure, enterprise-grade data intelligence platform designed to empower corporate banking teams with LLM-driven insights. This architecture leverages robust infrastructure, industry-standard access controls, and cutting-edge GenAI integration.

## 1. Architecture Highlights
* **LLM Engine:** Google AI Studio (models/text-embedding-004).
* **Data Layer:** PostgreSQL augmented with pgvector for knowledge base embeddings.
* **Transformations:** dbt for modular, testable data transformations and strict PII masking schemas.
* **Backend:** FastAPI for high-performance API endpoints.
* **Frontend:** Next.js for a responsive, intuitive chat interface.
* **Integration:** Model Context Protocol (MCP) enabling secure tool execution via the LLM.

## 2. Security & Industry Standards: The "Defense-in-Depth" Approach
Our implementation prioritizes rigorous security protocols suitable for the corporate banking sector.

### A. Strict Input Validation via Pydantic
All MCP tools (e.g., `sql_query`, `kb_search`) enforce strict data validation using Pydantic models. This ensures that inputs generated by the LLM are meticulously typed and validated before they interact with any internal backend logic, mitigating unexpected data payloads.

### B. SQL Injection Prevention
Direct string formatting for SQL is strictly prohibited. We employ **SQLAlchemy parameterized queries** (e.g., `text()` with bound parameters). The database engine explicitly treats variables as data rather than executable SQL logic.

### C. Zero-Trust Database Access (Least Privilege)
The database utilizes distinct roles. While data ingestion operates under robust permissions, the application queries the database via an `app_user` role.
Before executing dynamic SQL, the system enforces a strict environment:
```sql
SET ROLE app_user;
-- Execute query
RESET ROLE;
```
This guarantees the LLM only has **read-only** access scoped strictly to transformed, PII-masked tables (the `marts` schema).

### D. Robust Transaction Management
Every database interaction within our MCP tools is wrapped in robust `try/except/finally` blocks. Crucially, any exception triggers an explicit `session.rollback()`. This prevents hanging transactions and ensures a consistent database state, a non-negotiable standard in corporate banking systems.

## 3. Data Flow Overview
1. **User Query:** The Next.js frontend sends a query to the FastAPI backend.
2. **LLM Evaluation:** The backend interfaces with Google AI via the MCP server to determine the necessary tools (SQL lookup vs. KB search).
3. **Secure Execution:** The required tool executes securely (utilizing Pydantic validation and transaction rollbacks).
4. **Response Generation:** The LLM synthesizes the tool output into a cohesive response, returning it to the user.

## Conclusion
By embedding rigorous validation (Pydantic), transaction safety (explicit rollbacks), and multi-layered security (parameterization and least privilege), the Deep Insights Copilot stands as a production-ready asset aligned with the highest corporate banking standards.
